# Git版本管理与安全目录处理

## 目标

让资料库的重要结构、模板、脚本和内容变更可以被清晰提交，并避免把缓存、大文件或本地状态误提交。

## 触发条件

- 用户要求提交 commit、提供提交信息、查看 Git 状态或推送远程仓库。
- Git 报告 `dubious ownership` 或提示需要配置 `safe.directory`。
- 变更到达可独立保存点：阶段目标完成、必要检查通过，且用户没有表示仍要继续测试或继续改动。
- 修改了结构脚本、模板、工作流、知识点或索引。

## 执行步骤

1. 检查仓库根目录、当前分支、远程仓库和工作区状态。
2. 检查 `.gitignore` 是否覆盖缓存、本地状态和不应提交的大文件。
3. 运行结构检查脚本；若修改知识点元数据，先重新生成索引。
4. 查看将要提交的文件清单，确认没有 PDF、缓存或本地私有文件。
5. 到达保存点时判断是否需要更新 `CHANGELOG.md`：资料库级结构变化、脚本变化、工作流变化、大纲版本变化和批量知识点范围变化应记录；单个知识点小修、题目反馈入库和自动索引重建通常不记录。
6. 若需要更新 `CHANGELOG.md`，只写人需要知道的资料库级变化，不逐条罗列自动生成索引或每个知识点文件。
7. 若用户明确表示继续测试或继续改动，只把当前变更视为待保存点，不主动给提交信息。
8. 若用户要求提交、要求提交信息，或变更已到达可独立保存点，给出清楚的建议提交信息。
9. 按可独立理解的变更分批提交，提交信息使用清楚的中文或英文短句。
10. 提交后检查工作区状态。
11. 若配置了远程仓库，尝试推送；若没有远程仓库或权限失败，报告阻塞原因。
12. 若当前变更有必要 commit 保存但 AI agent 因权限、远程配置或其他环境限制无法完成提交，必须在最终回复末尾提醒用户手动 commit，并给出建议提交信息。

## 提交信息约定

提交信息使用 `类型(范围): 摘要` 格式；范围可省略。摘要使用中文短句，不写句号。

常用类型：

- `content`：正式知识点内容增量更新，例如 `content(操作系统): 补充页式管理易错点`。
- `feedback`：题目、错题、反馈或练习记录入库，例如 `feedback(线性表): 记录顺序表错题反馈`。
- `syllabus`：大纲版本、候选变化或大纲覆盖维护，例如 `syllabus(2027): 添加大纲变化候选报告`。
- `workflow`：新增或修改工作流，例如 `workflow: 新增废弃目录清理流程`。
- `agent`：整理 agent 资产、模板或维护约定，例如 `agent: 整理维护资产目录`。
- `script`：修改检查、索引或迁移脚本，例如 `script: 适配中文知识点目录`。
- `index`：重新生成自动索引且无其他源文件改动，例如 `index: 重新生成知识点索引`。
- `rename`：目录或文件重命名、移动，例如 `rename: 统一资料库目录命名`。
- `cleanup`：删除缓存、空目录或废弃旧路径，例如 `cleanup: 清理迁移后的旧目录`。
- `docs`：README 或说明文字调整，例如 `docs: 更新资料库目录说明`。
- `chore`：不改变资料库语义的杂项维护，例如 `chore: 更新忽略规则`。

一组提交同时涉及多类变更时，优先选择最能解释用户意图的类型；若变更较大，应拆分为多个提交。

## Git安全目录处理

当 Git 报告 `dubious ownership` 或提示当前仓库所有者与运行用户不一致时，按以下顺序处理：

1. 先确认当前目录确实是本资料库根目录，不要对未知路径添加信任。
2. 优先使用一次性参数查看状态，例如 `git -c safe.directory=E:/考研/408 status --short`。
3. 如果后续需要继续执行查看差异、分支、提交等 Git 命令，也优先在单次命令中追加同样的 `-c safe.directory=E:/考研/408` 参数。
4. 不要擅自修改全局 Git 配置；只有用户明确同意长期信任该路径时，才建议用户自行执行 `git config --global --add safe.directory E:/考研/408`。
5. 若一次性信任后仍无法读取状态，应报告具体错误，不要为了绕过问题更改文件所有权或回滚工作区。

## 禁止事项

- 不得为了提交而回滚用户已有改动。
- 不得提交 `.codex/`、`.agents/`、缓存目录或 Python 字节码。
- 不得默认提交大型 PDF 原始资料，除非用户明确要求。
- 不得手工修改自动生成索引来掩盖检查问题。
- 不得在无法完成 commit 时静默结束；必须明确提醒用户保存版本。

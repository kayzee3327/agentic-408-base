# 参考书可读性检查

## 目标

判断 `参考书/` 中的资料是否适合 AI agent 直接读取，并区分“可直接文本读取”“需要 OCR 或人工核验”“只能作为人工翻阅来源”。

## 触发条件

- 用户要求检查参考书、教材、PDF 或资料源是否适合 AI agent 读取。
- 新增、替换或迁移 `参考书/` 下的资料文件。
- 准备基于教材或教辅更新知识点事实前，需要确认资料能否可靠摘录。
- 发现 PDF 体积异常大、只有扫描页、缺少文本层、页码无法稳定定位或来源信息不清。

## 执行步骤

1. 先阅读 `AGENTS.md` 和 `agent/workflows/README.md`，确认是否还触发其他 active 工作流。
2. 列出 `参考书/` 下的文件路径、学科目录、格式、大小和修改时间。
3. 检查是否存在机器可读的来源说明，如书名、作者、版本、出版信息、用途优先级和页码口径；若缺失，在报告中标记为来源元数据不足。
4. 对 PDF 文件抽样检查文本层，优先抽查首页、中间页和末页；记录页数、书签目录、可提取文字量、中文或英文文本是否正常。
5. 将每份资料归类为：适合直接读取、可读但需抽查质量、文本很少不适合直接读取、基本不可文本读取、读取失败。
6. 对扫描型 PDF 或文本层很弱的资料，只能建议 OCR、人工核验或作为视觉参考；不得直接据此无来源生成具体知识事实。
7. 输出检查报告，说明总体结论、逐文件结论、主要风险和后续整理建议。
8. 本流程只检查资料可读性时，不修改正式知识点、题目、错题、大纲或索引。

## 禁止事项

- 不得把模板、书签、文件名或抽样失败的 PDF 当作正式知识事实来源。
- 不得因一本参考书不可读取而删除、覆盖或替换知识点 ID。
- 不得在没有教材、真题或可靠来源支撑时，把个人判断写入知识点核心事实。
- 不得手工维护 `索引/` 下的自动生成文件。

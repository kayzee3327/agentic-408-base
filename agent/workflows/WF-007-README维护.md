# README维护

## 目标

让根目录 `README.md` 只记录资料库的稳定入口信息，避免在普通小改、单个知识点维护或临时流程调整时频繁改动。

## 触发条件

- 项目一级目录、新增资料类型、模板体系、脚本体系或 agent 维护入口发生大变更。
- 工作流体系发生会影响新 agent 入门路径的大变更。
- 用户明确要求检查或修改 `README.md`。
- 执行任务时发现 `README.md` 中的目录、命令或维护入口已经明显过时。

## 执行步骤

1. 先判断当前任务是否属于项目级大变更；若只是单个知识点、题目、错题、反馈事件、索引重建或普通工作流小修，默认不检查、不修改 `README.md`。
2. 若触发条件成立，阅读根目录 `README.md`，只检查与本次大变更直接相关的章节。
3. 输出变更报告，说明触发原因、拟修改章节、是否涉及正式知识点内容。
4. 只做增量修改，保持 `README.md` 作为项目入口，不写长篇流程细节；可重复执行的流程细节应放入 `agent/workflows/`。
5. 若 README 修改源于新增或调整工作流，确保 `agent/workflows/README.md` 已同步索引。
6. 修改后运行结构检查脚本。

## 禁止事项

- 不得因为普通内容更新、单个文件小修或自动索引变化而主动修改 `README.md`。
- 不得把正式知识点事实、教材摘录、题目内容、错题复盘或个人思考写入 `README.md`。
- 不得把应由工作流承载的详细维护步骤搬进 `README.md`。
- 不得通过改写 `README.md` 来替代必要的模板、脚本、索引或工作流更新。

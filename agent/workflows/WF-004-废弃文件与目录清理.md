# 废弃文件与目录清理

## 目标

让资料库结构调整后的废弃文件、空目录、缓存和旧路径可以被安全清理，同时避免误删正式知识点、来源材料、ID登记和自动生成索引。

## 触发条件

- 用户要求重命名或移动目录；此类任务应视为“新路径确认 + 旧路径清理”的组合。
- 目录结构迁移后，旧目录、旧文件或旧缓存仍留在仓库中。
- 用户要求删除某个废弃文件或文件夹。
- Git 状态显示同一组变更中同时存在旧路径删除和新路径新增。
- 工作流、模板、脚本或说明文档迁移后，需要确认旧路径引用已经失效或替换。
- 发现 `__pycache__/`、`.pyc`、临时文件、空目录或不应提交的本地状态文件。

## 执行步骤

1. 先判定清理对象类型：正式知识点、来源材料、维护资产、自动生成索引、缓存、临时文件、空目录或本地状态文件。
2. 删除前输出清理报告，说明拟删除路径、废弃原因、是否已有新位置、是否影响正式知识点、是否需要更新引用，以及删除后要运行的检查。
3. 若清理对象是迁移后的旧路径，先确认新路径存在、内容可读取、相关脚本或工作流已改用新路径，再删除旧路径。
4. 删除非空目录前，必须逐项列出目录内容并确认每一项都属于可清理对象；不得直接对未知非空目录递归删除。
5. 空目录可以删除；缓存目录和缓存文件应优先删除，例如 `__pycache__/` 和 `*.pyc`。
6. 删除旧路径前后，搜索 README、AGENTS、工作流、模板和脚本中的旧路径引用，必要时同步更新；即使目录尚未创建，工作流中约定的未来目录名也要同步。
7. 批量替换路径后，确认文本和 JSON 文件仍为无 BOM UTF-8，避免脚本读取失败。
8. 若删除影响工作流、模板、脚本、状态文件、知识点、题目、错题、反馈事件或大纲版本，运行结构检查脚本。
9. 清理完成后检查 Git 状态，确认删除项和新增项能解释为同一组迁移或清理，不包含误删的正式内容。

## 禁止事项

- 不得把正式知识点页面、大纲版本、参考书摘录、题目记录、错题记录或反馈事件当作普通废弃文件直接删除。
- 不得删除或重建知识点ID登记内容来规避结构检查问题。
- 不得手工修改自动生成索引来掩盖删除造成的不一致。
- 不得在未确认内容的情况下递归删除非空目录。
- 不得为了让 Git 状态变干净而回滚或覆盖用户已有改动。

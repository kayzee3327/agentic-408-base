# 工作流索引

AI agent 处理本资料库任务时，应先阅读根目录 `AGENTS.md`，再阅读本索引，并根据任务触发条件阅读对应的 active 工作流。

## 使用规则

- 工作流用于规定可重复执行的资料库维护步骤。
- 流程、维护说明和可重复执行的规则默认放在 `agent/workflows/`；未经用户明确要求，不新建 `docs/` 目录。
- 工作流不得覆盖 `AGENTS.md` 中关于来源、ID、大纲、索引和结构检查的硬约束。
- 当任务涉及多个工作流时，先执行更基础的流程，再执行更具体的流程。
- 若现有工作流缺失、过时或无法覆盖正在执行的重复性任务，AI agent 应主动提出并增量更新工作流。

## 当前工作流

- `WF-001`：工作流维护，见 `agent/workflows/WF-001-工作流维护.md`。
- `WF-002`：做题反馈入库，见 `agent/workflows/WF-002-做题反馈入库.md`。
- `WF-003`：Git版本管理与安全目录处理，见 `agent/workflows/WF-003-Git版本管理与安全目录处理.md`；当用户要求提交、提交信息、检查 Git 状态、变更到达可独立保存点，或 Git 报告 `dubious ownership` / `safe.directory` 问题时，应阅读本工作流。
- `WF-004`：废弃文件与目录清理，见 `agent/workflows/WF-004-废弃文件与目录清理.md`；当目录迁移后存在旧路径、缓存、空目录或用户要求清理废弃文件夹时，应阅读本工作流。
- `WF-005`：参考书可读性检查，见 `agent/workflows/WF-005-参考书可读性检查.md`；当用户要求检查参考书、教材或 PDF 是否适合 AI agent 读取时，应阅读本工作流。
- `WF-006`：大纲版本更新，见 `agent/workflows/WF-006-大纲版本更新.md`；当用户提供新版大纲、要求比较大纲版本、维护 `大纲版本/` 或确认大纲变化时，应阅读本工作流。

## 更新后检查

新增或修改 `agent/workflows/` 下的工作流文件后，必须运行结构检查脚本。

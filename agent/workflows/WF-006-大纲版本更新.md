# 大纲版本更新

## 目标

让大纲更新以可追溯、可人工确认、可增量执行的方式进入资料库，避免覆盖整个知识库、误删旧知识点或随意创建新ID。

## 触发条件

- 用户提供新版考试大纲或要求保存新版大纲。
- 用户要求比较两个大纲版本的差异。
- 需要新增或修改 `大纲版本/versions.json`。
- 需要生成、阅读或处理 `大纲版本/<版本>/change_candidates.md`。
- 人工确认大纲条目的新增、删除、改名、移动、拆分或合并后，需要更新知识点页面范围状态。

## 执行步骤

1. 先确认新版大纲来源、版本名、保存位置和原文完整性；没有来源时不得写入正式知识点事实。
2. 将新版大纲原文或规范化副本保存到 `大纲版本/<版本>/raw.md`，并在 `大纲版本/versions.json` 中登记版本、标题、记录日期、来源和规范化副本路径。
3. 运行大纲比较脚本生成候选变化报告：

```bash
python agent/scripts/update_syllabus.py 大纲版本/<旧版本>/raw.md 大纲版本/<新版本>/raw.md --new-version <新版本>
```

4. 输出大纲变更报告，区分新增候选、删除候选、可能改名、移动、拆分和合并的条目。
5. 未经人工确认，不得根据候选报告直接创建、删除、改名或移动知识点。
6. 人工确认后再增量更新知识点：新增条目分配新ID；旧大纲中存在但当前大纲不再出现的知识点标记为 `deprecated` 或 `out_of_scope`，并保留原ID和已有引用。
7. 若修改知识点 YAML 元数据、移动知识点页面、新增或废弃知识点，先运行索引生成脚本，再运行结构检查脚本。
8. 若只保存新版大纲原文、更新版本记录或生成候选报告，运行结构检查脚本。

## 禁止事项

- 不得覆盖或重新生成整个知识库。
- 不得随意更改、复用或回收知识点ID。
- 不得把候选变化报告当作人工确认结果。
- 不得直接删除旧大纲条目对应的知识点页面。
- 不得无来源地补写具体知识事实。

# AGENTS.md

- 大纲决定知识库范围。
- 教材和真题决定准确性与深度。
- 禁止无来源地生成具体事实。
- 当前阶段不写长篇内容，不生成模拟题。
- 不得随意更改、复用或回收知识点ID。
- 大纲更新必须采用增量更新，不得覆盖或重新生成整个知识库。
- 新增、修改、删除大纲条目前，应先输出变更报告，再修改知识库。
- 模板规定页面结构，但模板文件不得被视为正式知识点、题目或错题。
- 索引是自动生成文件，不得手工维护。
- 修改知识点元数据后必须重新运行索引生成和结构检查脚本。
- 旧大纲中存在、当前大纲中不再出现的知识点不直接删除，应标记为 `deprecated` 或 `out_of_scope`，并保留原ID和已有引用。
- AI agent 处理非简单任务时，必须先阅读 `workflows/README.md`，并按触发条件阅读相关 active 工作流。
- 工作流文件位于 `workflows/`，用于记录可重复执行的资料库维护流程；工作流不得覆盖本文件中的硬约束。
- 未经用户明确要求，不得新建 `docs/` 目录；流程、维护说明和可重复执行的规则应优先写入 `workflows/`。
- 当任务暴露出可复用流程、现有流程过时或用户要求长期遵循某流程时，AI agent 应主动提出并增量更新工作流。
- 新增或修改工作流前，应先输出工作流变更报告；修改后必须运行结构检查脚本。
- 当 AI agent 判断当前变更有必要 commit 保存但无法自行提交时，必须在最终回复末尾提醒用户手动 commit。
- 做题反馈、错题复盘、刷题统计、截图转述、个人思考、教材摘录、教辅摘录和网络说法，必须先按输入类型归档，不得直接无来源写入正式知识点。
- 通过对话处理做题反馈或个人思考时，应先输出变更报告，说明输入类型、来源、关联知识点ID、拟新增或修改的文件、是否会修改正式知识点，再执行修改。
- 做题反馈只能增量更新知识点页面中的学习目标、基本解题方法、易错点、相关题目、自测、来源或修订记录，不得覆盖整个页面，不得更改或复用知识点ID。
- 个人思考可以作为个人反馈、解题动作或待验证项记录；若缺少教材、真题或可靠来源支撑，不得写成核心知识事实。
- 完整题目优先使用题目模板；已经做错、半会半不会或暴露稳定弱点的输入优先使用错题模板；非完整题目的反思、统计和观察使用反馈事件模板。
- 修改题目、错题、反馈事件或知识点后，必须运行结构检查脚本；若修改知识点元数据，还必须重新运行索引生成脚本。
